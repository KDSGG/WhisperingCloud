# 支付与安全接入清单

## 1. 支付架构
- 客户端不保存支付密钥。
- 由服务端创建支付会话（如 Stripe Checkout Session）。
- 通过支付平台 Webhook 二次确认订单状态，不以前端回调作为支付成功依据。

## 2. 基础防护
- 全站 HTTPS。
- 开启 CSP、HSTS、X-Content-Type-Options、Referrer-Policy。
- 表单输入校验与输出转义，避免 XSS。
- 对下单与登录接口做限流和 IP 风险控制。

## 3. 数据安全
- 订单与用户信息分表，最小权限访问。
- 敏感字段加密存储（手机号、地址）。
- 日志脱敏，避免记录明文敏感数据。

## 4. 订单与风控
- 每笔支付请求附带唯一 nonce，防重放。
- 订单状态机：`PENDING -> PAID -> IN_PRODUCTION -> SHIPPED`。
- 对高风险交易增加人工复核。

## 5. 合规
- 公示退款、售后、隐私政策。
- 按目标市场遵守数据合规要求（如中国网络安全法、GDPR 等）。
